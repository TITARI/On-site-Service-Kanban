# 展商数据看板与智能表格导入设计

## 1. 背景

现有后台“展览数据”只支持上传 `.xlsx`、`.xls` 或 `.csv`，并从第一张工作表读取固定字段。实际项目工作簿通常包含多个结构不同、内容不完整的工作表，例如：

- `普通绿色搭建汇总`
- `标展楣牌`

每个项目的工作表名称、表头和字段完整度可能变化。系统不应把工作表行直接当成业务任务，也不应永久保存与现场协同无关的原始列。新功能需要把多个工作表统一转换为项目级“系统展商”，并提供可查询、编辑和分配现场搭建成员的数据看板。

本次样本文件为 `第23届中原农资双交会后勤表.xlsx`。其中目标工作表可转换为约 272 个系统展商。样本中存在同一展位对应多个展商的情况，因此展位号不能单独作为唯一键。

## 2. 目标

1. 管理员先创建或选择展览项目，再上传一个或多个本地工作簿。
2. 从任意数量的工作表中识别系统需要的展商字段。
3. 使用确定性规则优先、高智商 AI 辅助、管理员最终确认的方式完成字段映射与数据归一化。
4. 以 `项目 + 展位号 + 展商` 唯一识别系统展商。
5. 支持同一展位存在多个展商。
6. 支持展商与现场搭建组成员的多对多关联。
7. 支持项目内增量同步、字段级差异确认和管理员手工编辑。
8. 让现有工单准确关联具体展商，而不是只关联展位号。

## 3. 非目标

- 不直接连接或同步腾讯文档、企业微信文档。
- 不把 Excel 每一行建成独立工单或设计任务。
- 不永久保存无关工作表字段、原始行内容、备注或沟通记录。
- 不保存“统一状态”或“风险”作为展商业务字段。
- 不根据表格自动创建系统用户。
- 不让重新导入覆盖人工分配的现场搭建成员。

## 4. 核心业务模型

### 4.1 展览项目

管理员必须先创建或选择展览项目，再执行导入。不同项目的数据相互隔离。

项目至少包含：

- 项目 ID
- 项目名称
- 状态
- 开始时间和结束时间，可选
- 创建时间和更新时间

### 4.2 系统展商

系统展商是看板的核心实体。唯一身份为：

```text
项目 ID + 规范化展位号 + 规范化展商名称
```

允许：

- 同一个展位号对应多个展商。
- 同一个展商在不同展位或不同项目出现。

系统展商只保存以下业务字段：

| 字段 | 说明 |
| --- | --- |
| 展位号 | 主要查询与关联依据之一 |
| 展商 | 展商全称 |
| 位置 | 楼层与展馆合并，如 `一楼 / 1E`；缺少楼层时只保存展馆 |
| 面积数值 | 规范化后的平方米数值，可为空 |
| 面积规格 | 如 `9×4m`，可为空 |
| 类型 | 归一化类型，管理员可修改或新增分类 |
| 销售 | 销售人员姓名，可为空 |

面积在界面中组合显示，例如：

```text
36㎡ · 9×4m
```

无法识别数值或规格时，允许只保存其中一项。

初始类型归一化集合包括：

- 普通绿搭
- 升级绿搭
- 普标
- 精标
- 其他/待确认

类型集合可由管理员维护。

### 4.3 现场搭建成员关联

“搭建商”在系统中不是自由文本字段，而是现场搭建组的具体成员。

系统展商与成员为多对多关系：

- 一个成员可以负责多个展商或展位。
- 一个展商可以绑定多个成员。

导入时：

1. 优先通过手机号匹配现场搭建组成员。
2. 无手机号时通过规范化姓名匹配。
3. 匹配失败时进入人工分配，不自动创建用户。
4. 没有搭建商列的工作表，导入后由管理员批量分配成员。

重新导入时保留系统内已有的人工成员分配。

## 5. 导入架构

导入采用“分阶段导入 + 标准展商库”方案。上传文件不会直接修改正式展商数据。

### 5.1 导入阶段

1. 管理员创建或选择展览项目。
2. 上传一个或多个 `.xlsx`、`.xls` 或 `.csv` 文件。
3. 服务端解析所有工作表的名称、有效区域、表头和单元格值。
4. 规则引擎识别常见表头和明确字段。
5. 高智商 AI 对未知表头、类型、面积及规格进行辅助判断。
6. 管理员确认工作表选择和字段映射。
7. 系统生成标准化候选展商。
8. 系统与项目内现有展商进行比较。
9. 管理员预览并处理新增、字段差异、缺失记录和成员匹配失败。
10. 管理员提交后，系统在事务中写入正式数据。

### 5.2 字段映射

规则优先匹配常见表头及其变体，例如：

- 展位号
- 公司名称、企业名称、展商
- 楼层、所属展馆、展馆
- 面积、规格
- 方案类型、展位类别
- 销售人员、业务员
- 搭建商

AI 只处理规则不能可靠识别的映射，不替代确定性规则。

字段映射预览必须展示：

- 来源文件
- 来源工作表
- 识别出的表头行
- 系统字段
- 来源列
- 识别方式：规则、AI 或人工
- AI 置信度和简短理由
- 样例值

管理员确认后的映射可以保存为可复用的导入模板。后续相似表格优先使用已确认模板，再由规则和 AI 补充。

### 5.3 AI 生效规则

- 高置信度结果自动进入导入预览，但不绕过管理员最终提交。
- 中低置信度结果必须人工确认。
- AI 不可用、超时或配置缺失时，自动降级到规则识别和手动映射。
- AI 不负责自动创建用户、删除展商或覆盖人工数据。
- 导入预览保留 AI 判断依据和置信度；正式展商库不保存无关原始内容。

### 5.4 数据归一化

归一化过程包括：

- 去除展位号和展商名称首尾空白。
- 统一全角、半角及常见空白字符。
- 位置由楼层和展馆合并。
- 面积拆分为平方米数值和规格说明。
- 类型映射到系统分类。
- 搭建商文本解析为一个或多个姓名、手机号候选。

不得仅因公司名称包含“原名称”“更名”等备注性内容而擅自拆分或改写展商。无法确定时进入人工确认。

## 6. 增量同步与冲突

同一项目重新上传表格时，系统按 `项目 + 展位号 + 展商` 匹配。

### 6.1 新增记录

表格中存在、系统中不存在的记录显示为新增。管理员确认后创建系统展商。

### 6.2 已有记录

匹配到已有展商时逐字段比较：

- 值相同：无需处理。
- 系统为空、表格有值：建议采用表格值。
- 系统有值、表格不同：显示字段级差异。
- 管理员手工修改过的字段：仍显示差异，不自动覆盖。

管理员可以逐字段或批量选择：

- 采用表格值
- 保留系统值

### 6.3 表格中消失的记录

系统中存在、当前导入来源中未出现的记录进入“缺失记录确认区”，不自动删除或停用。管理员明确确认后才能停用。

一个项目可以有多个来源文件或工作表，因此“缺失”判断必须限定在可识别的来源范围内，不能因为某一张工作表未包含其他类型展商而误报整个项目数据。

### 6.4 重复数据

- 完全相同的 `项目 + 展位号 + 展商` 候选合并为一条，并展示重复来源。
- 同展位不同展商分别保留。
- 同展商不同展位分别保留。
- 缺少展位号或展商名称的候选不能写入正式库，必须修复或跳过。

## 7. 临时数据与保留策略

系统只把标准化业务字段写入正式展商库。

导入期间可以临时保存：

- 文件名和文件摘要
- 工作表名
- 表头映射
- 标准化候选值
- 差异结果
- AI 置信度和判断理由
- 解析错误

导入完成或取消后：

- 删除上传的原始文件内容。
- 删除无关列和原始单元格内容。
- 删除无需审计的临时候选数据。
- 保留导入任务统计、操作人、结果数量和映射模板。

系统不得将原工作簿备注、到款情况、沟通记录等无关内容复制到正式展商记录。

## 8. 展商数据看板

看板沿用现有后台绿色视觉体系，桌面端采用高密度数据表格，窄屏采用卡片化布局。

### 8.1 页面结构

1. 项目选择器
2. 项目数据概览
3. 搜索与筛选工具栏
4. 导入差异入口
5. 展商数据表格
6. 批量操作栏
7. 展商详情抽屉

概览指标包括：

- 系统展商数
- 已分配搭建成员数
- 待分配搭建成员数
- 待确认导入差异数

这些是页面统计，不是展商字段。

### 8.2 表格列

- 选择框
- 展位号
- 展商
- 位置
- 面积
- 类型
- 销售
- 现场搭建成员
- 操作

同一展位有多个展商时分行展示，并提供文字提示，不能只通过颜色表达。

### 8.3 搜索与筛选

支持搜索：

- 展位号
- 展商名称
- 销售
- 现场搭建成员

支持筛选：

- 位置
- 类型
- 成员分配状态
- 现场搭建成员

### 8.4 批量操作

管理员选择多个展商后，可以：

- 批量分配一个或多个现场搭建成员
- 批量修改类型
- 批量停用，必须二次确认

### 8.5 详情与编辑

点击“查看”打开右侧详情抽屉。抽屉展示七项基础数据和搭建成员关系。

管理员可以：

- 编辑七项基础数据
- 添加或移除搭建成员
- 查看最近一次导入变化

现场搭建成员只能查看自己负责的展商，不可编辑基础数据。

## 9. 工单关联

现有工单流程需要从只依赖展位号，升级为关联具体系统展商。

### 9.1 匹配规则

- 展位号只匹配一个展商：自动关联。
- 展位号匹配多个展商：暂停自动建单，提示提交人选择具体展商。
- 提交人无法完成选择：进入管理员复核。
- 未匹配到展商：保留现有补充展位信息流程，不虚构展商。

### 9.2 工单快照

工单保存：

- 展商 ID
- 展位号快照
- 展商名称快照

后续管理员修改展商基础数据时，不追溯改写历史工单快照。

## 10. 权限

### 后台管理员

- 创建和维护展览项目
- 上传工作簿
- 确认字段映射和 AI 建议
- 处理导入差异
- 编辑展商基础数据
- 分配现场搭建成员
- 停用展商

### 现场搭建成员

- 查看自己负责的展商
- 查看相关工单
- 不可导入文件
- 不可修改展商基础数据
- 不可修改其他成员分配

### 普通成员

- 查询可见项目的展商
- 提交工单时选择具体展商
- 不可导入、编辑或分配

所有写操作必须经过现有后台身份验证和权限校验，不能只依赖前端隐藏按钮。

## 11. 数据库设计方向

在现有 `exhibitions` 和 `exhibition_booths` 基础上演进：

### exhibition_booths

保留表名以降低迁移范围，但语义调整为系统展商。需要支持：

- 同一项目相同展位号的多个展商
- 面积数值
- 面积规格
- 位置
- 类型
- 销售
- 启用状态
- 字段更新时间或人工修改标记

现有自由文本 `builder` 不再作为正式业务字段使用，成员关系迁移到关联表。现有 `raw_payload` 不用于保存新导入工作簿的无关原始列；历史值可在迁移完成并验证后清理。

唯一索引从：

```text
exhibition_id + booth_number
```

调整为：

```text
exhibition_id + normalized_booth_number + normalized_company_name
```

### exhibition_booth_builders

新增展商与现场搭建成员的关联表：

- exhibition_booth_id
- person_id
- assigned_by
- assigned_at

以 `exhibition_booth_id + person_id` 唯一。

### import_jobs

扩展现有导入任务，记录：

- 项目 ID
- 文件名
- 文件摘要
- 导入状态
- 新增数
- 更新数
- 保留系统值数
- 缺失待确认数
- 跳过数
- 操作人

### import_mapping_templates

保存已确认的工作表识别和字段映射模板。模板只保存结构特征和映射，不保存业务数据。

### tickets

新增可空的 `exhibition_booth_id`，并保留现有展位号和公司名称字段作为快照。

## 12. API 与服务边界

建议拆分为以下服务：

- `WorkbookInspectionService`：读取工作簿、发现工作表和表头。
- `FieldMappingService`：规则映射、AI 补充和模板匹配。
- `ExhibitorNormalizationService`：生成标准化候选展商。
- `ImportDiffService`：比较新增、变化、缺失和冲突。
- `ExhibitorImportService`：事务提交正式数据。
- `ExhibitorAssignmentService`：管理展商与搭建成员关系。
- `ExhibitorResolutionService`：为工单按展位号解析具体展商。

导入 API 采用分阶段接口，避免一次请求完成所有操作：

1. 创建导入任务并上传文件。
2. 获取工作表与映射建议。
3. 提交确认后的字段映射。
4. 获取标准化预览和差异。
5. 提交字段级决策。
6. 事务应用导入。

所有阶段必须校验项目权限和导入任务所有权。

## 13. 错误处理

- 文件过大：服务端流式接收并限制文件大小，返回明确限制。
- 文件格式错误：指出文件名和解析失败原因。
- 无有效工作表：允许管理员返回重新选择或上传。
- 缺少必填字段：指出工作表、来源列和示例行。
- AI 不可用：降级到规则和人工映射。
- 搭建成员匹配失败：保留展商导入，进入成员待分配队列。
- 字段冲突未决：禁止提交受影响字段，但允许跳过对应候选。
- 事务失败：回滚正式数据，不产生半成品。
- 重复提交：导入提交接口必须具备幂等保护。

## 14. 测试策略

### 单元测试

- 表头别名规则
- 位置合并
- 面积数值和规格解析
- 类型归一化
- 展位号和展商名称规范化
- 搭建成员手机号、姓名匹配
- 字段级差异计算

### 集成测试

- 一个工作簿包含多个有效工作表
- 多个工作簿导入同一项目
- 未知表头由 AI 建议
- AI 不可用时降级
- 同展位多个展商
- 相同展商不同展位
- 重新导入保留人工成员分配
- 手工字段与表格字段冲突
- 缺失记录不被自动删除
- 事务失败完整回滚

### 页面测试

- 项目选择
- 搜索和筛选
- 表格与窄屏卡片布局
- 批量分配成员
- 详情抽屉编辑
- 导入映射确认
- 差异逐项和批量决策
- 权限限制
- 键盘操作、焦点状态和非颜色提示

### 工单测试

- 单一展商自动关联
- 多展商要求选择
- 管理员复核
- 工单保存展商 ID 和名称快照

## 15. 验收标准

1. 管理员可以创建或选择项目，并上传包含多个工作表的本地工作簿。
2. 系统能够通过规则和 AI 识别七项展商数据及搭建成员候选。
3. 管理员可以确认字段映射和所有中低置信度建议。
4. 正式展商库只保存已定义的业务字段和成员关联。
5. 同一展位的多个展商可以分别存在、查询和编辑。
6. 重新导入不会直接覆盖人工修改，也不会覆盖人工搭建成员分配。
7. 字段差异可以逐项或批量确认。
8. 工作表中消失的记录不会自动删除。
9. 没有搭建商字段的数据可以在看板中批量分配成员。
10. 工单遇到同展位多个展商时必须选择具体展商。
11. AI 不可用时仍可通过规则和人工映射完成导入。
12. 导入提交失败时正式数据保持不变。
