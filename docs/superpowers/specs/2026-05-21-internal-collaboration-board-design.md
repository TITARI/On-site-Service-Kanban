# 内部协同看板（移动端）设计文档

## 1. 背景与目标

本阶段仅建设“内部协同看板”子项目，服务现场执行团队，目标是用移动端实现统一工单协同与处置闭环。

核心目标：

1. 全员可提交/查看全量工单，管理员可派单与配置规则。
2. 支持文本+图片反馈，支持工单追加信息与帖式回复。
3. 同一展位号下，基于语义识别重复问题，避免重复建单。
4. 重复问题按催单处理，并按问题类型设置催单允许时间。
5. 处理链路可追踪：受理时间、处理人、当前进度、处理回执。

本阶段明确不包含：

1. 微信/企业微信入口打通（第二阶段接入）。
2. AI+H5展商服务中心其他模块（导航、会刊等）。

## 2. 范围与业务规则

### 2.1 工单状态流

固定状态流：

`待受理 -> 处理中 -> 挂起 -> 已解决 -> 已关闭`

规则：

1. 新建工单初始为`待受理`。
2. 派单并接手后进入`处理中`，记录受理时间。
3. 挂起必须填写原因与预计恢复时间（可选）。
4. 已解决后可进入已关闭，保留全量历史轨迹。

### 2.2 主视图组织方式

工单主看板按“问题类型”分组展示，支持状态筛选和关键词检索。

### 2.3 重复问题与催单规则

判定前提：

1. 必须同一展位号。
2. 同一展位号下，再由AI做文本语义相似判定。

处理策略（混合）：

1. 高置信度：直接合并到原工单，转为催单。
2. 中置信度：进入人工确认池，由管理员判定“合并催单”或“新建工单”。
3. 低置信度：直接新建工单。

催单控制：

1. 每个问题类型配置催单允许时间。
2. 未达到时限不转催单；达到时限后可触发催单并升级优先级。

## 3. AI策略（方案C）

采用“AI优先 + 向量检索增强”的策略，并要求双模型接入：

1. 快速AI：低延迟，用于高频实时判断（初判）。
2. 高智商AI：高准确，用于复杂判断、复核、超时分析。

### 3.1 场景路由

1. 问题类型自动归类：默认快速AI，失败或低置信度时回退高智商AI/人工确认。
2. 语义判重：快速AI初判，高智商AI复核（按阈值触发）。
3. 超时增强：触发高智商AI + 向量检索，生成相似案例与处置建议。

### 3.2 超时策略（标准版）

超时后执行：

1. 推送“相似案例+建议步骤”给处理人/管理员。
2. 自动升级催单优先级。
3. 不自动改工单状态（状态仍由人确认变更）。

### 3.3 可观测与可追溯

每次AI判定需记录：

1. 使用模型与版本。
2. 请求耗时、置信度。
3. 判定动作（新建/催单/人工确认）。
4. 回退路径（例如主模型失败切换备用）。

## 4. 自动派单策略

派单链路：

1. 一级：按`展位号 + 问题类型`自动匹配责任人。
2. 二级：自动失败进入管理员派单或处理组认领。

说明：匹配规则由管理员维护，可按展位区段、问题类型、责任人优先级配置。

## 5. 基础主数据与表格导入

展位主数据由管理员通过表格导入维护，至少包含：

1. 展位号
2. 公司名称
3. 业务员
4. 搭建商

建议支持格式：`.xlsx` / `.csv`。

导入规则：

1. 展位号作为主键，执行增量更新（upsert）。
2. 必填字段缺失时阻止导入并返回错误行。
3. 支持“预检模式”先校验后提交，避免脏数据落库。

应用规则：

1. 工单提交时通过展位号自动关联公司信息。
2. 工单标题由系统自动生成：`展位号 + 公司简称 + 问题类型`。
3. 若主数据缺失公司简称，临时回退为公司名称截断或“未知公司”标记。

## 6. 工单数据模型（逻辑）

### 6.1 工单主字段

1. 基础：工单ID、展位号、公司名称、公司简称、问题描述、图片列表、问题类型、提交人、提交时间。
2. 流程：状态、受理时间、处理人、责任组、最后更新时间、受理后耗时（实时计算）。
3. AI：归类模型、判重模型、判重置信度、判定结果、AI建议内容。
4. 催单：催单次数、最近催单时间、催单等级、关联原工单ID。
5. 超时：超时阈值快照、是否触发超时分析、分析时间。

### 6.2 多人反馈与回复

1. 同问题可多人反馈，记录反馈用户列表与反馈时间。
2. 详情页支持帖式回复（类似帖子楼层），支持文字+图片。
3. 回复角色标签：反馈人、处理人、管理员、系统AI。
4. 工单处理完成后，向所有反馈用户发送处理结果回执（本阶段先站内，微信阶段接入外部触达）。

## 7. 优先级与排序规则

列表按综合优先级排序：

1. 问题严重度（问题类型优先级 + 风险词权重）。
2. 催单次数。
3. 受理后时长。

同分处理：

1. 先看最近催单时间。
2. 再看提交时间（更早者优先）。

## 8. 移动端信息架构

1. 提交页：展位号、描述、图片上传、问题类型（默认“自动”）。
2. 列表页：按问题类型分组，支持状态筛选与排序。
3. 详情页：进度、处理人、受理时间、时间线、帖式回复区、追加反馈入口。
4. 我的页：我提交的、我参与的、待我确认（管理员可见人工确认池）。
5. 管理页：问题类型配置、催单时限、AI模型映射、派单规则、主数据导入。

## 9. 权限模型

1. 全员：查看全量工单、提交工单、追加信息、参与回复。
2. 管理员：派单、处理人工确认池、配置问题类型/催单时限/AI路由/派单规则、主数据导入。

## 10. 异常兜底

1. AI不可用：降级为规则判定 + 人工复核，不阻塞建单。
2. 向量服务失败：不阻塞流转，仅缺失建议，记录告警。
3. 派单失败：转管理员派单或组内认领。
4. 图片上传失败：允许先建文字工单，后续补图。
5. 快速AI与高智商AI冲突：按配置走“高智商优先”或“人工确认优先”。

## 11. 监控指标与验收标准

### 11.1 监控指标

1. 时效：平均受理时长、平均处理时长、超时工单数。
2. 质量：已解决率、已关闭率、重复建单率、误判率。
3. 压力：催单量、升级催单占比、各类型积压量。
4. AI：自动归类命中率、人工复核占比、模型失败回退率。
5. 反馈：处理结果回执触达率、触达延时。

### 11.2 第一版验收标准

1. 移动端完成提交、列表、详情、回复、追加反馈、状态流转。
2. 问题类型默认“自动”，且管理员可配置至少1个非自动选项。
3. 同展位号语义重复可触发“合并催单/人工确认/新建”分流。
4. 标准版超时策略生效：推荐+提醒+优先级升级。
5. 自动派单优先按`展位号 + 问题类型`，失败后可人工派单/认领。
6. 表格导入可维护展位主数据并驱动工单标题自动生成。

## 12. 开发边界与后续阶段

本设计对应第一阶段的内部协同看板。后续阶段将扩展：

1. 微信/企业微信消息入口与回执出站。
2. 展商端AI+H5服务中心功能集成。
