# 工单归档与自动验收设计

## 目标

将已闭环工单从日常待办列表中收纳到统一归档区域，并在业务组长时间未进行人工验收时自动完成验收闭环。自动验收结果通过现有微信/企微出站消息队列同步给业务反馈侧和认领处理侧。

## 范围

- 移动端工单列表增加“归档工单”折叠窗口。
- `已关闭` 工单统一进入归档窗口，其他状态继续按问题类型分组展示。
- 后台系统配置页面增加自动验收开关和自动验收时效（分钟）。
- `已解决` 工单超过配置时效且仍未人工验收时，自动变更为 `已关闭`。
- 自动验收写入工单时间线，并分别通知业务反馈侧与认领处理侧。
- 不新增独立定时任务服务。

## 配置

在 `AppConfig` 中增加：

```ts
autoAcceptance: {
  enabled: boolean;
  timeoutMinutes: number;
}
```

默认启用，默认时效为 30 分钟。后台系统配置页面提供：

- “启用自动验收”复选框。
- “处理完成后自动验收时效（分钟）”数字输入框。
- 时效限制为 1 至 10080 分钟（7 天）。

配置保存继续使用现有 `/api/admin/config`，并存入 `app_config_versions.config_json`，不需要新增配置表或数据库迁移。

## 自动验收判定

自动验收服务接收当前时间和配置，筛选满足以下条件的工单：

1. 自动验收已启用。
2. 工单状态为 `已解决`。
3. 从工单最后一次进入 `已解决` 的时间起，已经达到配置时效。

“进入已解决时间”优先从时间线中寻找最后一条包含“状态变更为已解决”的记录；历史数据没有对应时间线时，回退到 `updatedAt`。

自动验收时：

- 状态改为 `已关闭`。
- `updatedAt` 更新为自动验收执行时间。
- 时间线新增 `receipt` 记录：
  - 内容：`业务组在 N 分钟内未验收，系统已自动验收通过并关闭工单`
  - 操作人：`系统`
- 重复执行时不再次关闭、不重复写时间线、不重复发送通知。

## 触发机制

不引入新的常驻定时任务。自动验收在现有服务入口执行：

- 移动端或后台读取工单列表时。
- 读取单个工单详情时。
- 微信/企微桥接领取待发送出站消息时。

这样即使无人打开前端，正在运行的消息桥接轮询也会推动自动验收和通知；如果桥接未运行，下一次系统读取工单数据时仍会补执行。

自动验收必须在 MariaDB 事务内完成。数据库更新使用带状态条件的更新或事务内锁定，确保多个请求同时触发时只有一个执行者完成闭环和排队通知。

## 通知

自动验收成功后排队两类消息：

1. 业务反馈侧
   - 优先发送到工单来源会话或反馈人微信身份。
   - 内容说明业务组超时未验收、系统已自动验收并关闭。

2. 认领处理侧
   - 发送到 `assignmentGroup`，没有分组时使用 `handlerName`。
   - 内容说明工单已自动验收闭环。

两条消息均写入现有 `outbound_messages`，关联 `relatedTicketId`。若业务侧和处理侧最终解析为同一目标，仍保留两条带不同语义的通知，便于双方知道各自角色下的闭环结果。

## 前端归档交互

`TicketList` 将输入工单拆分为：

- 活跃工单：状态不是 `已关闭`。
- 归档工单：状态是 `已关闭`。

活跃工单保持当前问题类型分组。归档区位于列表底部：

- 使用原生 `<details>` / `<summary>`，默认折叠。
- 标题显示“归档工单”和归档数量。
- 展开后按更新时间倒序显示已关闭工单。
- 归档工单仍可点击进入现有详情页。
- 没有归档工单时不显示归档区域。
- 只有归档工单、没有活跃工单时，页面仍显示归档窗口，不显示“暂无工单”。

## 后台界面

自动验收设置放在后台“系统配置”页面，作为独立配置卡片，靠近微信/企微消息设置：

- 标题：`自动验收`
- 简短说明：处理组标记完成后，业务组在时效内未验收将由系统自动闭环。
- 启用开关。
- 分钟数输入。
- 独立保存按钮和保存反馈。

## 错误处理

- 配置分钟数不是整数或超出范围时，后台页面阻止提交并显示明确错误。
- API 配置校验再次限制范围，避免绕过前端。
- 自动验收单个工单失败时事务回滚，不产生半完成通知。
- 已经人工验收、退回处理或被其他请求自动验收的工单不再处理。

## 测试

- 配置校验：默认值、合法分钟数、关闭功能、越界拒绝。
- 自动验收服务：未到时效不处理、到时关闭、时间线记录、幂等。
- MariaDB：自动验收持久化并排队业务侧和处理侧通知。
- 列表组件：已关闭工单只出现在归档折叠区，活跃列表不包含关闭工单。
- 后台组件：显示配置、保存合法值、拒绝非法值。
- API/仓库入口：列表、详情和出站消息领取前会执行自动验收。
- 完整测试与生产构建。

## 非目标

- 不增加新的工单状态“已归档”。
- 不提供手工归档按钮。
- 不实现独立任务调度器、Cron 服务或后台进程。
- 不改变人工验收通过和验收退回的现有权限及流程。
